# 컨텍스트 
기존의 props로 전달하는 방식 대신 컴포넌트 트리를 통해 바로 컴포넌트에 전달하는 방식이다.

여러 컴포넌트에서 계속 접근이 일어날 수 있는 데이터들이 있는 경우 사용하는 것이 좋다.
(반대로 재사용성은 떨어진다.)

### 예시
```js
    const ThemeContext = React.createContext('light');

    function App(props) {
        return (
            <ThemeContext.Provider value="dark">
                </Toolbar>
            </ThemeContext.Provider>
        )
    }

    function Toolbar(props) {
        return (
            <div>
                <ThemedButton />
            </div>
        )
    }

    function ThemedButton(props) {
        return (
            <ThemeContext.Consumer>
                {value => <Buttton theme={value}>/ } //  가장 상위의 value값인 dark를 받는다.
            </ThemeContext.Comsumer>
        )
    }
```

## 컨텍스트 API
### React.createContext()
컨텍스트를 생성하는 역할을 한다. ()안에는 기본값.

### Context.Provider
해당 컴포넌트로 하위 컴포넌트들을 감싸게 되면 모든 하위 컴포넌트들이 해당 컨텍스트에 접근할 수 있게 된다.

### Class.contextType
provider 하위에 있는 클래스 컴포넌트에서 컨텍스트의 데이터에 접근하기 위해 사용하는 것이다.

### Context.Consumer
컨텍스트의 데이터를 구독하는 컴포넌트이다. (함수 컴포넌트의 경우 이걸 쓴다)

### Context.displayName
리액트 개발자 도구에 표시되는 속성

## useContext
함수 컴포넌트에서 컨텍스트를 쉽게 사용할 수 있다.

```js
    function MyComponent(props) {
        const value = useContext(MyContext); //파라미터로 컨텍스트 객체를 넣어줘야 한다.

        return (...)
    }
```
