# 훅
함수 컴포넌트에 클래스 컴포넌트와 같이 state를 정의하거나 생명주기에 맞춰 코드를 실행시킬 수 있도록 하는 것

훅은 이름 앞에 use를 붙여 나타낸다.

## useState 
state를 사용하기 위한 훅.

### 예시

```js
import React, {userState} from "react"

function Counter(props) {
    var count = 0;

    return (
        <div>
        <p>
            총 {count} 번 클릭했습니다.
        </p>
        <button onClick={() => count++}>
            클릭
        </button>
        </div>
    )
}
```
위 코드는 재렌더링 하기 않아 변경된 값이 화면에서 보여지지 않는다.
이럴 경우 state를 이용해 업데이트 해주어야 한다.

const [변수명, set변수명] = useState(초기값);

```js
import React, {userState} from "react"

function Counter(props) {
    const [count, setCount] = useState(0);

    return (
        <div>
        <p>
            총 {count} 번 클릭했습니다.
        </p>
        <button onClick={() => setCount(count + 1)}>
            클릭
        </button>
        </div>
    )
}
```

useState()를 사용하는 방법에서는 변수 각각의 set함수가 존재하게 된다.

## useEffect
사이드 이펙트(서버에서 데이터 가져오거나 수동으로 DOM 변경 등)를 수행하기 위한 훅.

useEffect()는 클래스 컴포넌트에서의 함수인 componentDidMount(), componentDidUpdate(), componentWillUnmount()와 동일한 기능을 하나로 통합헤서 제공한다.

### 예시
useEffect(이펙트 함수, 의존성 배열);

의존성 배열의 값이 하나라도 변경될 경우 이펙트 함수가 실행된다.

```js
    import React, {useState, useEffect} from "react";

    function Counter(props) {
        const [count, setCount] = useState(0);

        useEffect(() => {
            //document title은 브라우저 창에 표시되는 문자열
            document.title = `총 ${count}번 클릭했습니다.`
        })

        return (
            <div>
                <p>총 {count}번 클릭했습니다.</p>
                <button onClick={() => setCount(count + 1)}>
                클릭
                </button>
            </div>
        )

    }
```
위의 예시는 의존성 배열이 없는데, 이 경우 리액트는 DOM이 변경된 이후 해당 이펙트 함수를 실행하도록 받아들인다.

매번 랜더링 될때마다 실행되는 구조로 결과적으로 보면 componentDidMount(), componentDidUpdate()와 동일한 역할인 셈이다.


```js
    import React, {useState, useEffect} from "react";

    function UserStatus(props) {
        const [isOnline, setIsOnline] = useState(null);

        function handleStatusChange(status) {
            setIsOnline(status.isOnline);
        }

        useEffect(() => {
            ServerAPI.subscribeUserStatus(props.user.id, handleStatusChange);
            return() => {
                ServerAPI.unsubscribeUserStatus(props.user.id, handleStatusChange);
            }
        });

        if(isOnline === null) {
            return "loading...";
        }

        return isOnline ? 'online' : 'offline';
    }
```

useEffect()에서 리턴하는 함수는 컴포넌트가 마운트 해제될 때 호출된다.
결과적으로 클래스 컴포넌트의 componentWillUnmount() 의 역할을 한다고 볼 수 있다.


## useMemo
Memoized Value(이미 계산된 결과를 저장해두었다 같은 입력에 대해 저장된 값을 재사용하는 것)을 리턴하는 훅.

파라미터로 create 함수와 의존성 배열을 받으며 배열 중 변수가 변했을 경우에만 새로 create 함수를 호출하여 결과값을 반환한다.

```js
    const memoizedValue = useMemo(
        () => {
            return computeExpensiveValue(의존성 변수1, 의존성 변수2)
        },
        [의존성 변수1, 의존성 변수2]
    )
```

컴포넌트가 다시 렌더링될 때마다 연산량이 높은 작업을 재실행하는 것을 피할 수 있다.
useMemo() 안의 함수는 렌더링이 일어나는 동안 실행된다는 점을 기억해야 한다.
(렌더링이 일어나는 동안이 아닌건 useEffect() 함수 사용)


## useCallBack
useMemo와 비슷한 역할을 하나, 값이 아닌 함수를 리턴한다.

## useRef
레퍼런스(특정 컴포넌트에 접근할 수 있는 객체)를 사용하기 위한 훅.

```js
    const refContainer = useRef(초기값);
```

파라미터로 들어온 초기값으로 초기화된 레퍼런스 객체를 반환한다.
레퍼런스 객체에는 .current 속성이 있는 현재 참조하고 있는 앨리먼트를 의미한다.

```js
    function TextInputWithFocusButton(props) {
        const inputElem = useRef(null);

        const onButtonClick = () => {
            inputElem.current.focus();
        };

        return(
            <>
                <input ref={inputElem} type="text"/>
                <button onClick={onButtonClick}>Focus the input</button>
            </>
        )
    }
```

버튼 클릭 시 호출되는 함수에서 inputElem.current를 통해 실제 엘리먼트에 접근해 focus() 함수를 호출하고 있다.

## 훅의 규칙
1. 훅은 무조건 최상위 레벨에서만 호출해야 한다. (매번 같은 순서로 호출되어야 함)
2. 리액트 함수 컴포넌트에서만 훅을 호출해야 한다.
